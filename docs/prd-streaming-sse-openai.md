# PRD: Стриминг ответов чата (SSE) через Next.js API и OpenAI

## Контекст проекта

- **Стек:** Next.js 16 (App Router), React 19, TypeScript.
- **Чат:** `src/modules/chat/components/Chat.tsx` — локальный стейт сообщений, `onSendMessage` прокидывается наружу.
- **Точка входа:** `src/app/page.tsx` — сейчас заглушка (`console.log`), комментарий про подключение бэкенда.
- **Существующие API:** `src/app/api/conversations/route.ts` и др.
- 

## Цель

Реализовать отображение ответа ассистента **потоком** (символы/фрагменты по мере генерации), не раскрывая API-ключ OpenAI в браузере. Сервер проксирует поток от OpenAI к клиенту в формате, совместимом с **Server-Sent Events (SSE)**.

## Не в scope (явно)

- Хранение диалогов в БД (можно добавить позже).
- Настоящий `EventSource` с GET для тела истории чата (см. раздел про SSE).

---

## 1. Уточнение: «SSE» и ограничение EventSource

- **Классический `EventSource` в браузере** в основном поддерживает **GET**. Передать полную историю чата в теле POST через `EventSource` нельзя без обходных путей.
- **Рекомендуемый вариант для чата:** тот же **wire-format SSE** (`data: ...\n\n`), но доставка — `**fetch` с `POST` + чтение `response.body` через `ReadableStream`** и ручной разбор событий.
- Альтернатива «чистого» EventSource: отдельный сценарий (сохранить диалог POST → стримить GET по `id`) — избыточно для первой версии.
**Вывод для PRD:** принимаем контракт «POST `/api/chat` + поток `text/event-stream` + парсинг SSE на клиенте».

## 4. Сервер: Route Handler стриминга

**Предлагаемый путь:** `src/app/api/chat/route.ts` (рядом с существующими `src/app/api/**/route.ts`).


| Шаг                                                                                                                                                             | Требование                                                                                                                                   |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| Метод                                                                                                                                                           | `POST` (тело — JSON с историей сообщений)                                                                                                    |
| Тело запроса                                                                                                                                                    | Например: `{ "messages": [ { "role": "user" | "assistant" | "system", "content": string } ] }` — финальная схема фиксируется при реализации  |
| Вызов OpenAI                                                                                                                                                    | `chat.completions.create` с `stream: true`, указать `model` и `messages`                                                                     |
| Ответ клиенту                                                                                                                                                   | `new Response(stream, { headers })`, не буферизовать весь текст на сервере перед отправкой                                                   |
| Заголовки ответа                                                                                                                                                | `Content-Type: text/event-stream`, `Cache-Control: no-cache`, `Connection: keep-alive`; при прокси при необходимости `X-Accel-Buffering: no` |
| Формат чанков                                                                                                                                                   | SSE: для каждого фрагмента текста строка вида `data: <JSON или текст>\n\n` (единый формат на всём проекте)                                   |
| Ошибки                                                                                                                                                          | При ошибке до/во время стрима — осмысленный HTTP-статус и/или финальное SSE-событие об ошибке (на усмотрение реализации)                     |
| **Runtime:** по умолчанию Node для Route Handler обычно совместим с официальным SDK; при использовании `export const runtime = 'edge'` проверить совместимость. |                                                                                                                                              |

## 5. Клиент: получение потока и обновление UI

**Файл:** минимум `src/app/page.tsx` (или вынесенный хук, напр. `useChatStream`).


| Шаг                                                                                                                                                                                                                                                                                                     | Требование                                                                                                                    |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| Триггер                                                                                                                                                                                                                                                                                                 | По отправке сообщения пользователем                                                                                           |
| История                                                                                                                                                                                                                                                                                                 | Собрать массив `messages` для API (включая новое сообщение пользователя)                                                      |
| Плейсхолдер ассистента                                                                                                                                                                                                                                                                                  | Сразу добавить сообщение с `role: "assistant"` и пустым `content`                                                             |
| Запрос                                                                                                                                                                                                                                                                                                  | `fetch('/api/chat', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ messages }) })` |
| Ошибки                                                                                                                                                                                                                                                                                                  | Если `!response.ok` — показать ошибку, не парсить тело как успешный стрим                                                     |
| Чтение                                                                                                                                                                                                                                                                                                  | `response.body.getReader()` + `TextDecoder`                                                                                   |
| Парсинг                                                                                                                                                                                                                                                                                                 | Накопление буфера, разделение по `\n\n`, обработка строк `data: ...`                                                          |
| Обновление стейта                                                                                                                                                                                                                                                                                       | Дописывать полученные куски в `content` **последнего** сообщения ассистента                                                   |
| **Согласование с `Chat.tsx`:** сейчас `messages` живут внутри `Chat`. Для стриминга нужно либо **поднять стейт в `page.tsx`** и передавать в `Chat` как props, либо расширить контракт `Chat` (например, controlled messages + callback). Иначе родитель не может обновлять текст ассистента по чанкам. |                                                                                                                               |

## Критерии приёмки

1. Пользователь отправляет сообщение — ответ ассистента **появляется и удлиняется по мере генерации**, без ожидания полного текста.
2. `OPENAI_API_KEY` **не** попадает в клиентский бандл и **не** виден в DevTools как часть запроса к OpenAI (только вызов к своему `/api/chat`).
3. При ошибке API пользователь видит понятную обратную связь, а не «молчание» или бесконечную загрузку.
4. Повторная отправка во время стрима заблокирована или обработана осмысленно (на усмотрение UX).
